# LPE 0.2.0

## Current State/Functionality Overview

`0.2.0` identifies the Exchange-compatibility scope for `EWS`, guarded `MAPI over HTTP`, and Outlook Anywhere / RPC over HTTP routing. The release scope keeps canonical mailbox, contacts, calendar, tasks, submission, and `Sent` behavior in core `LPE`.

## Implementation/Usage

- `EWS` runs in `crates/lpe-exchange`.
- `MAPI over HTTP` routes are authenticated:
  - `/mapi/emsmdb`
  - `/mapi/nspi`
- Outlook Anywhere / RPC over HTTP uses:
  - `/rpc/rpcproxy.dll`
- Exchange-style surfaces must use canonical `LPE` state.
- Client `SMTP` is not implemented inside the core service.
- `LPE-CT` custody and spool recovery are release gates.
- Autodiscover publication remains gated by explicit configuration.
- `IMAP` remains a supported mailbox-access compatibility path.
- Protocol publication is gated by `docs/architecture/0.2.0-protocol-depth-gates.md`.

## Reference Table/List

| Surface | Rule |
| --- | --- |
| `JMAP` | state/change/query/push consistency over canonical state |
| `IMAP` | Outlook and Thunderbird compatibility transcripts |
| `ActiveSync` | mobile/iOS compatibility lab |
| `EWS` | bounded canonical mailbox/contact/calendar/task compatibility |
| `MAPI over HTTP` | authenticated Outlook desktop Exchange-account route |
| `/rpc/rpcproxy.dll` | Outlook Anywhere / RPC over HTTP path |
| `SMTP` | stays in `LPE-CT` |
| `LPE-CT` custody | durable spool custody and recovery gate |
| `Sent` | canonical `LPE` copy |

Publication readiness evidence for the `0.2.0` release gates is recorded in `docs/architecture/0.2.0-protocol-depth-gates.md`. The evidence covers local protocol crate tests, LPE-CT custody and recovery tests, plus live JMAP, ActiveSync, EWS, MAPI over HTTP, and RPC proxy readiness checks against the configured test deployment.
