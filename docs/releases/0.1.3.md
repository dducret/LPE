# LPE 0.1.3

## Current State/Functionality Overview

`0.1.3` identifies the Exchange-compatibility scope for `EWS`, guarded `MAPI over HTTP`, and Outlook Anywhere / RPC over HTTP routing. The release scope keeps canonical mailbox, contacts, calendar, tasks, submission, and `Sent` behavior in core `LPE`.

## Implementation/Usage

- `EWS` runs in `crates/lpe-exchange`.
- `MAPI over HTTP` routes are authenticated:
  - `/mapi/emsmdb`
  - `/mapi/nspi`
- Outlook Anywhere / RPC over HTTP uses:
  - `/rpc/rpcproxy.dll`
- Exchange-style surfaces must use canonical `LPE` state.
- Client `SMTP` is not implemented inside the core service.
- Autodiscover publication remains gated by explicit configuration.
- `IMAP` remains a supported mailbox-access compatibility path.

## Reference Table/List

| Surface | Rule |
| --- | --- |
| `EWS` | bounded canonical mailbox/contact/calendar/task compatibility |
| `MAPI over HTTP` | authenticated Outlook desktop Exchange-account route |
| `/rpc/rpcproxy.dll` | Outlook Anywhere / RPC over HTTP path |
| `SMTP` | stays in `LPE-CT` |
| `Sent` | canonical `LPE` copy |
