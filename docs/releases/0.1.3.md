# LPE 0.1.3

`0.1.3` is the EWS implementation release.

It moves the desktop Exchange compatibility focus from the `IMAP` bootstrap work
completed through `0.1.2` to the `EWS` adapter in `crates/lpe-exchange`.

The release also removes the historical PostgreSQL migration chain and replaces
it with a single canonical initialization script for fresh installs.

Breaking change:

- legacy updates are not supported
- a fresh install and database recreation are required
- startup now fails unless the database reports schema version `0.1.3`

EWS scope:

- Exchange-style clients can discover and use the published `EWS` endpoint when an administrator enables EWS autodiscovery
- `EWS` reuses canonical mailbox, contacts, calendar, draft, submission, and deletion paths
- `MAPI`, `RPC`, client `SMTP`, and any Exchange-specific mailbox store remain out of scope
- `IMAP` remains a supported mailbox-access communication protocol and the `0.1.2` compatibility foundation; it receives correctness fixes, but it is no longer the main `0.1.3` development driver
