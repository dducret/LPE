# LPE 0.1.3

`0.1.3` is the EWS plus full MAPI/Outlook implementation release.

It moves the desktop Exchange compatibility focus from the `IMAP` bootstrap work
completed through `0.1.2` to the `EWS` adapter in `crates/lpe-exchange` and
the classic Outlook for Windows `MAPI over HTTP` path.

The release also removes the historical PostgreSQL migration chain and replaces
it with a single canonical initialization script for fresh installs.

Breaking change:

- legacy updates are not supported
- a fresh install and database recreation are required
- startup now fails unless the database reports schema version `0.1.3`

Exchange and Outlook scope:

- Exchange-style clients can discover and use the published `EWS` endpoint when an administrator enables EWS autodiscovery
- `EWS` reuses canonical mailbox, contacts, calendar, tasks, draft, submission, and deletion paths
- full classic Outlook for Windows support over `MAPI over HTTP` is a release goal, behind authenticated `/mapi/emsmdb` and `/mapi/nspi` routes
- supported `mapiHttp` Outlook desktop publication is gated until profile creation, first sync, day-two sync, `NSPI` name resolution, canonical send/draft flows, reconnect behavior, and authoritative `Sent` visibility pass the interoperability matrix
- legacy `RPC`, Outlook Anywhere, client `SMTP` inside the core service, and any Exchange-specific mailbox store remain out of scope
- `IMAP` remains a supported mailbox-access communication protocol and the `0.1.2` compatibility foundation; it receives correctness fixes, but it is no longer the main `0.1.3` development driver
