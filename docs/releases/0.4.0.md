# LPE 0.4.0

`0.4.0` starts from a fresh database schema and supports bounded, documented
SQL compatibility updates for initialized `0.4` deployments.

## Upgrade Policy

- New core `LPE` databases must be empty before applying the `0.4.0-sql-v2`
  schema with `installation/debian-trixie/init-schema.sh`.
- Existing initialized `0.4` databases can be advanced by
  `installation/debian-trixie/update-lpe.sh` when the repository includes a
  documented, idempotent SQL compatibility patch for the current schema version.
- `update-lpe.sh` checks the installed schema version before running any SQL
  compatibility patch, then rebuilds and redeploys code and web assets.
- Use `LPE_RESET_SCHEMA=true` with `init-schema.sh` only for an intentional
  destructive reset of a disposable or explicitly rebuilt deployment.

## Operator Notes

- After initialization, `check-lpe.sh` verifies that the installed database
  reports schema version `0.4.0-sql-v2` and includes any documented compatibility
  columns required by the current code.
- Current compatibility patch: `update-lpe.sh` makes
  `mapi_navigation_shortcuts.target_folder_id` nullable for Common Views group
  header rows and adds `group_header_id` plus `group_name` for Outlook
  `WunderBar` shortcut group/header persistence.
- The same patch adds the canonical recoverable-items store, account/mailbox
  retention and legal-hold columns, and change-log/tombstone constraints needed
  before MAPI dumpster ROP behavior is exposed. It also repairs and verifies
  the recoverable-item replay-shape constraint so restored and purged recovery
  rows can carry valid source mailbox membership identifiers.
- Bounded MAPI search-folder criteria now include audited
  `RopSetSearchCriteria` / `RopGetSearchCriteria` parity for the documented
  canonical subset, including round-trippable `PidNameKeywords` category
  equality and parseable errors for rejected MS-OXOSRCH shapes.
- The compatibility patch repairs active MAPI identity material from each
  persisted global counter and enforces active source-key uniqueness so Outlook
  sync/import identity rows cannot retain duplicate `PidTagSourceKey` values.
- MAPI identity startup repair also removes orphaned content sync checkpoints
  and associated configuration rows whose backing mailbox or folder identity no
  longer exists.
- Incremental MAPI hierarchy sync now emits retained mailbox tombstones as
  deleted folder IDs so Outlook can remove cached folders instead of keeping
  stale hierarchy entries.
- Incremental MAPI hierarchy sync now includes user-saved search-folder
  definition changes and retained search-folder tombstones so Outlook can add,
  update, and remove cached search folders without leaving ghost entries.
- MAPI object/source-key identity lookup now rejects stale dynamic mailbox,
  search-folder, and associated-configuration identities before they can be
  reopened from an old Outlook cache.
- MAPI associated-configuration delete sync now reports only retained MAPI
  identity rows and no longer allocates or reactivates identity state while
  exporting deleted view/config objects.
- MAPI Inbox associated-configuration lookup now returns Outlook's exact
  `IPM.Sharing.Index` virtual row and canonicalizes the observed Outlook
  sharing named-property aliases used by that row.
- MAPI Outlook `NamedView` projections now return the minimal view descriptor
  payload through the Outlook-associated `0x0e0b0102` binary property instead
  of advertising an empty view body.
- MAPI get-props diagnostics now decode `PidTagDefaultViewEntryId` message
  targets so Outlook view failures can be tied to the exact folder/view object
  referenced by the client.
- MAPI Inbox associated-configuration `FindRow` now pins exact virtual Outlook
  config hits to a matching follow-up table restriction so subsequent
  `QueryRows` calls see the same row model.
- MAPI Outlook `NamedView` FAI rows now expose the official MS-OXOCFG
  `PidTagViewDescriptorName` and canonical `PidTagViewDescriptorVersion`
  properties in addition to Outlook's private 0x683x view metadata.
- MAPI Outlook `NamedView` FAI rows now emit MS-OXOCFG view-definition
  descriptor rows and descriptor strings for the default Compact and Sent To
  mail views instead of the previous blank-column-only placeholder stream.
- MAPI content sync now includes associated view/config changes and deletes for
  Contacts, Calendar, and Tasks folders instead of treating those folders as
  item-only streams.
- MAPI Common Views navigation shortcuts now write durable change-log rows and
  replay through content sync so Outlook Favorites and shortcut state do not
  require a full resync after create, update, or delete.
- MAPI navigation shortcut identity lookup and startup repair now reject stale
  shortcut identities whose Common Views row no longer exists, preventing
  deleted shortcuts from being reopened from an old Outlook cache.
- A missing or weak `LPE_INTEGRATION_SHARED_SECRET` degrades readiness and
  pauses the outbound bridge worker, but the core administration API still
  starts so operators can sign in and repair configuration.
