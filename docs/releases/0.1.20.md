# LPE 0.1.20

## Current State/Functionality Overview

`0.1.20` identifies the repository baseline for the documented `LPE` and `LPE-CT` architecture. The baseline keeps the split core/sorting-center model, canonical state rules, and supported adapter set aligned across Rust crates, web packages, and installation guidance.

## Implementation/Usage

- Core `LPE`:
  - canonical mailboxes
  - contacts
  - calendars
  - tasks
  - search
  - rights
  - submission
  - webmail
  - administration API
- `LPE-CT`:
  - inbound `SMTP`
  - outbound relay
  - authenticated client submission where deployed
  - quarantine
  - accepted-domain policy
  - DKIM signing
  - local technical stores
- Supported adapters:
  - `JMAP Mail`
  - `JMAP Contacts`
  - `JMAP Calendars`
  - `JMAP Tasks`
  - `IMAP`
  - `ActiveSync`
  - `EWS`
  - guarded `MAPI over HTTP`
  - `Sieve` / `ManageSieve`
  - `CardDAV`
  - `CalDAV`
  - DAV `VTODO`
- Observability:
  - structured tracing
  - Prometheus-compatible metrics
  - health/readiness checks
