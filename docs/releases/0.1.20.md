# LPE 0.1.20

`0.1.20` is a repository alignment release.

It does not introduce a new product scope. It consolidates the current documented and implemented MVP baseline into a coherent release marker across Rust crates, web packages, installation guidance, and top-level documentation.

## Included Scope Already Present in the Repository

- persistent administration console backed by `PostgreSQL`
- canonical message submission with authoritative `Sent` handling, protected `Bcc` metadata, and outbound queueing toward `LPE-CT`
- explicit internal integration contract between `LPE` and `LPE-CT`
- MVP adapters for `JMAP Mail`, `JMAP Contacts`, `JMAP Calendars`, `IMAP`, `ActiveSync`, `Sieve` / `ManageSieve`, `CardDAV`, and `CalDAV`
- canonical personal tasks integrated into the account workspace
- web client backed by persistent account authentication and mailbox/workspace APIs
- public autoconfiguration for `Thunderbird` and minimal `Outlook` autodiscovery for `ActiveSync`
- initial observability with metrics and structured tracing
- Debian Trixie installation and update scripts for `LPE` and `LPE-CT`

## Release Notes

- aligned workspace, `LPE-CT`, and web package versions to `0.1.20`
- removed duplicated French Markdown sections and kept the maintained documentation in English
- consolidated license text, policy, and accepted exceptions into root `LICENSE.md`
- tightened `AGENTS.md` so agents read only the global architecture and task-relevant documentation
- removed the redundant `LPE-CT/README.md` in favor of the dedicated architecture and operations documentation
